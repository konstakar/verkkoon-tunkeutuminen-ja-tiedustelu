# a) ARP hyökkäys

- ei antanut ajaa oletetusti linkeissä olevaa ARP-hyökkäystä, mutta ylipäätänsä ARP-hyökkäys: hyökkääjä lähettää vääriä ARP (address resolution protocol) vastauksia, hyökkääjä esim. väittää olevanmsa reititin tai "X" ip-osoite.

# b) ICMP spoofing ja TCP session hijacking

- sama ongelma

- ICMP spoofing: internet control message protocol spoofing eli lähdeosoitteen väärentäminen.
- hyökkääjä lähettää ICMP-paketteja, väärentää sen lähde IP:n ja vastaanottaja luulee viestin tulevan luotettavalta taholta.

- TCP session hijacking: transmission control protocol
- kaappaa session, asiakas <-> palvelin TCP-yhteys
- hyökkääjä seuraa liikennettä, ennustaa/lukee sekvenssinumeron, lähettää väärennettyjä paketteja. palvelin luulee oikeaksi asiakkaaksi.

# c) TCP syn flood

- sama ongelma...
- TCP: asiakas -> palvelin (syn), palvelin -> asiakas (syn-ack), asiakas -> palvelin (ack)
- hyökkäys: lähettää suuren määrän syn-paketteja (floodaa) palvelimelle, väärennetyistä osoitteista (spoofing), ei lähetä viimeistä ack-vastausta
